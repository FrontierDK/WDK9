Remarks

Parsed XQuery expressions are useful because they allow the cost of query parsing and optimization to be amortized over many evaluations.

There are two basic forms of the execute method: one that takes a Value object, and another that does not. For methods that do not take a Value, the XQuery must restrict the scope of the query using either the collection() or the doc() XQuery navigation functions, or an exception is thrown when accessing the context item.

For those forms of the execute method that take a Value, the value is bound to the context item, and can be accessed using the "." notation in XQuery.

This class implements the IDisposable interface, and the Dispose method must always be called on it to allow the clean up of unmanaged memory and container handles.
