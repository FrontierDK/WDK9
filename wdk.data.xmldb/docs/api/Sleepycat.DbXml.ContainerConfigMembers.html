Public Instance Constructors

ContainerConfig Constructor Creates a Environment.Configuration object with default values set.

Public Instance Properties

AllowValidation When loading documents into the container, validate the XML if it refers to a DTD or XML Schema.
Checksum Do checksum verification of pages read into the cache from the backing filestore.
ContainerType Set whether documents are broken down into their component nodes, and these nodes are stored individually in the container, or stored intact with all white space and formatting is preserved. NodeContainer is preferred.
Create If the container does not currently exist, create it.
DirtyRead Causes the container to Support dirty reads; that is, read operations on the container may return data that has been modified by other transactions but which has not yet been committed.
Encrypt Encrypt the database using the cryptographic password specified to DbEnv::set_encrypt.
Exclusive Return an error if the container already exists.
IndexNodes Causes the indexer to create index targets that reference nodes rather than documents.
Mode The file system permissions mode of the container.
NoMMap Do not map this container into process memory (see the DbEnv::set_mp_mmapsize method for further information).
ReadOnly Open the container for reading only.
Threaded Cause the container handle to be free-threaded; that is, concurrently usable by multiple threads in the address space.
Transactional Cause the container to support transactions.
XACreate Instead of creating a standalone database, create a database intended to be accessed via applications running under an X/Open conformant Transaction Manager.
