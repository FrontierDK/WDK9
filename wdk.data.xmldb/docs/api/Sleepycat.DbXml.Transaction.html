Remarks

Methods in the Transaction class are used to abort and commit the transaction. Transaction handles are provided to Container, Manager, and other objects that query and modify documents and containers in order to transactionally protect those database operations.

Transaction objects are instantiated using Manager.CreateTransaction.

Transaction handles are not free-threaded; transactions handles may be used by multiple threads, but only serially, that is, the application must serialize access to the Transaction handle. Once the Abort or Commit methods are called, the handle may not be accessed again, regardless of the method's return. In addition, parent transactions may not issue any operations while they have active child transactions (child transactions that have not yet been committed or aborted) except to call Abort and Commit.

This class implements the IDisposable interface, and the Dispose method must always be called on it to allow the clean up of unmanaged memory and file handles, and the flushing of any outstanding database data.

If Abort or Commit is called on the transaction, Dispose is automatically called. Calling Dispose before aborting or committing the transaction while result in the transaction being aborted. This mechanism can be used in combination with "using" blocks, to make blocks of code using the transaction exception safe.

Note    This class is analogue to the C++ class XmlTransaction.
