Remarks

The result of a query is a collection of Value objects. An empty Results object can be created using Manager.CreateResults.

A Results object is usually created by executing a query. There are several ways that a query is performed. One is to call Manager.Query directly. This mechanism is appropriate for one-shot queries that will not be repeated.

A second approach is to create an QueryExpression using Manager.Prepare. You then execute the query expression using QueryExpression.Execute. This approach is appropriate for queries that will be performed more than once as it means that the expense of compiling the query can be amortized across multiple queries.

Note that when you perform a query, you must provide a QueryContext object. Using this object, you can indicate whether you want the query to be performed eagerly or lazily. If eager evaluation is specified (the default), then the resultant values are stored within the Results object. If lazy evaluation is selected, then the resultant values will be computed as needed. In this case the Results object will maintain a reference to the affected containers (Container), query context (QueryContext), and expression (QueryExpression).

The Results class implements the IEnumerator interface, as well as adding the MovePrevious method, to allow backwards iteration for eagerly evaluated Results objects. The Add method and Size property also require the Results object to have been evaluated eagerly.

This class implements the IDisposable interface, and the Dispose method must always be called on it to allow the clean up of unmanaged memory and container handles.

Note    This class is analogue to the C++ class XmlResults.
